📚 EduPlanApp: Система Управління Освітніми Завданнями

EduPlanApp — це мінімалістична система управління завданнями (To-Do List), спеціально розроблена для студентів та освітніх проектів. Вона дозволяє організовувати навчальні завдання, використовуючи систему пріоритетів та категорій.

🚀 Технології та Функціонал

Технології

Фреймворк: ASP.NET Core MVC (на основі мінімалістичного хостингу).

ORM: Entity Framework Core (EF Core).

База Даних: Крос-провайдерна підтримка (SQLite, PostgreSQL, MS SQL Server, In-Memory).

Аутентифікація: Налаштована через Google OAuth та локальний вхід (на прикладі).

Функціонал

Проект на поточному етапі надає наступну функціональність:

Управління завданнями (TaskItem): Створення, перегляд, редагування та видалення завдань.

Категоризація: Призначення завдань до довідникових категорій (Лабораторні роботи, Теоретичні предмети тощо).

Пріоритизація: Встановлення рівня важливості (Високий, Середній, Низький).

Синхронізація БД: Автоматичне застосування міграцій та заповнення початковими даними (Seed) при старті програми.

🛠️ Налаштування та Запуск

1. Конфігурація бази даних

Проект використовує ключ DatabaseType у файлах конфігурації для вибору провайдера.

Базова конфігурація (appsettings.json):

{
  "DatabaseType": "Sqlite", 
  "ConnectionStrings": {
    // ... всі основні рядки підключення
  }
}


Перезапис для розробки (appsettings.Development.json):
Якщо ви використовуєте PostgreSQL на нестандартному порту або з власними обліковими даними, створіть (або оновіть) файл appsettings.Development.json для перезапису налаштувань.

Приклад appsettings.Development.json:

{
  "ConnectionStrings": {
    "PostgresConnection": "Host=localhost;Port=4717;Database=eduplanappdb_postgres;Username=postgres;Password=NICER312"
  }
}


Це гарантує, що ваші локальні налаштування не потраплять у базовий файл конфігурації.

2. Створення універсальної міграції

Видаліть папку Migrations (якщо вона містить прив'язані до Postgres файли).

Створіть нову міграцію:

dotnet ef migrations add InitialMigration


3. Запуск програми

Проект автоматично застосовує міграції та заповнює початковими даними при кожному старті завдяки логіці в Program.cs.

dotnet run
